# Word of the Day

> ⚠️ **이 프로젝트는 비상업적 팬 카피(clone)입니다.**
> 구체적으로는, 카카오톡 채팅방 미니게임 목록에서 **2026년 7월 5일 기준 서비스 중인 "오늘의 단어"** 게임을
> **무제한 웹 버전(셔플 모드)** 으로 재구현한 것입니다. 하루 한 단어가 아니라 원하는 만큼 계속 플레이할 수 있습니다.
> Wordle 및 "오늘의 단어"의 아이디어·디자인에 대한 권리는 각 원저작자(The New York Times, 카카오 등)에게 있으며,
> 본 저장소는 학습·재미 목적의 개인 프로젝트로 어떠한 상업적 의도도 없습니다.

단어 맞추기 게임 **[Wordle](https://www.nytimes.com/games/wordle/index.html)** 의 한국어 버전입니다.
정답은 5개의 **기본 자모**(자음·모음)로 이루어지며, 매 판 후보 단어 중 하나가 랜덤으로 출제됩니다.

일반 두벌식 자판과 달리 복합 자모를 입력받지 않습니다. 겹자음·이중모음은 기본 자모의
조합으로 분해해서 다룹니다. (예: `ㄲ` → `ㄱ`+`ㄱ`, `ㅐ` → `ㅏ`+`ㅣ`)

- `개미` → `ㄱ ㅏ ㅣ ㅁ ㅣ`
- `까치` → `ㄱ ㄱ ㅏ ㅊ ㅣ`
- `어둠` → `ㅇ ㅓ ㄷ ㅜ ㅁ`

## 단어 후보

정답 후보는 국립국어원의 **한국어 학습용 어휘 목록**에서 추립니다.

- 출처: [국립국어원 — Korean Learners' Vocabulary List](https://www.korean.go.kr/front_eng/down/down_02V.do?etc_seq=71&pageIndex=1)

원본(`data/korean_words.txt`, EUC-KR)에서 아래 조건으로 후보를 생성합니다.

1. 품사가 정확히 `명`(명사)
2. 단어에서 숫자를 제거(동음이의어 접미사, 예: `가격03` → `가격`) 후 중복 제거
3. 자모로 분해했을 때 정확히 5개
4. `data/blacklist.txt`에 있는 단어는 제외 (부적절하거나 빼고 싶은 단어를 한 줄에 하나씩)

결과는 난이도 등급(A/B/C)과 함께 `data/whitelist.txt`(`단어<TAB>등급`)로 저장되며,
게임은 현재 **A등급**만 정답 후보로 사용합니다. (난이도는 `index.html`의 `GRADE`로 조정)

```bash
node build_whitelist.js   # data/korean_words.txt → data/whitelist.txt, words.js 재생성
```

## 파일 구성

```
word_of_the_day/
├─ index.html          게임 UI + 상호작용 (로직/단어는 외부에서 로드)
├─ logic.js            decompose / evaluate (+테스트용 WORDS)  ← 게임·테스트 공유 단일 소스
├─ logic.test.js       로직 테스트
├─ build_whitelist.js  korean_words.txt → whitelist.txt / words.js 생성
├─ words.js            등급별 후보 임베드 (자동 생성, 브라우저용)
├─ data/
│  ├─ korean_words.txt 원본 어휘 목록 (국립국어원, EUC-KR)
│  ├─ blacklist.txt    후보에서 제외할 단어 목록
│  └─ whitelist.txt    후보 단어 + 등급 (자동 생성)
└─ README.md
```

## 실행 방법

별도 빌드나 서버가 필요 없습니다. 브라우저로 `index.html`을 열면 됩니다.

```bash
# macOS
open index.html
```

또는 파일 탐색기에서 `index.html`을 더블클릭해도 됩니다.

### 조작

- **입력**: 화면 자판을 클릭하거나 물리 키보드를 사용합니다.
  물리 키보드는 한글 IME on/off와 무관하게 **두벌식 키 위치**로 인식합니다.
  (예: `R` → `ㄱ`, `K` → `ㅏ`)
- **삭제**: `⌫` 키 또는 `Backspace`
- **제출**: `결과보기` 버튼 또는 `Enter`
- 총 **5번**까지 시도할 수 있습니다.

### 색상 규칙

- 🟩 초록: 자모와 위치가 모두 정답
- 🟨 노랑: 정답에 있지만 위치가 다름
- ⬜ 회색: 정답에 없음

같은 자모를 여러 번 입력한 경우, 정답에 포함된 개수만큼만 색이 칠해집니다.
초록(정답 위치)을 우선 배정하고, 남는 개수는 앞쪽 위치부터 노랑으로 칠하며, 초과분은 회색입니다.

## 테스트 방법

로직(`logic.js`)에 대한 테스트는 [Node.js](https://nodejs.org)로 실행합니다.

```bash
node logic.test.js
```

- 모든 테스트를 통과하면 종료 코드 `0`, 하나라도 실패하면 `1`을 반환합니다. (CI 연동 가능)
- 테스트 항목: 자모 분해, 정답 후보 길이, 기본 채점, 중복 자모 규칙(초록 우선 → 앞 글자 노랑 → 초과분 회색).

로직을 수정할 때마다 이 명령으로 회귀 검증하세요. 게임과 테스트가 `logic.js` 하나를 공유하므로,
테스트가 통과하면 게임의 채점 동작도 동일하게 보장됩니다.
